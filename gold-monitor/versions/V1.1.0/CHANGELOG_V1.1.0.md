# V1.1.0 版本更新说明

更新日期：2026-03-09

本次版本基于 V1.0.0 做了稳定性、可维护性和数据源可用性增强，重点如下：

1. 数据源链路增强
- `GC近月` 新增东方财富主源，当前顺序为 `Eastmoney -> Sina -> Yahoo`。
- `GLD` 新增东方财富主源，当前顺序为 `Eastmoney -> Yahoo -> Stooq`。
- `UUP` 新增东方财富主源，当前顺序为 `Eastmoney -> Yahoo -> Stooq`。
- 保留多源回退策略，单个来源失败时继续尝试后备来源。

2. 数据缺失补全能力增强
- 抓取逻辑改为“主源给主价格，后备源继续补缺字段”，尤其会继续补 `成交量`。
- 支持字段级合并，不再因为主源缺少成交量就直接保留空值。

3. 来源诊断能力增强
- 新增诊断文件 `out/source_diagnostics_latest.txt`。
- 每轮会记录 `XAU/USD`、`USD/CNY`、`GC近月`、`GLD`、`UUP/DXY代理`、`10年实际利率` 的实际命中来源。
- 当某字段由后备源补全时，会在诊断信息中体现。

4. 历史数据稳定性增强
- `jsonl` 历史文件读取加入坏行容错。
- 当历史文件中存在损坏行时，程序会自动跳过并回写清洗后的内容，避免单条坏记录影响后续跟踪。

5. 文件写入稳定性增强
- 采集存储与查看快照分离，主 CSV 被占用时仍能继续记录历史。
- 保留快照回退机制，输出文件被占用时会写入 `snapshots` 目录。
- 守护进程日志写入改为追加模式，降低写日志时的文件冲突风险。

6. 代码结构整理
- 收敛重复的 HTTP 请求逻辑，统一为可复用的文本/JSON 请求函数。
- 拆分状态构造、CSV 行构造、CSV 文档生成、来源摘要等小函数。
- 输出状态对象中的来源摘要已净化，只保留对排查有用的公开字段。

7. 测试与验证
- 测试覆盖扩大到东方财富期货/美股解析、多源合并、来源摘要判断等关键路径。
- 当前测试脚本 `test/monitor.test.mjs` 已通过 23 项断言。
- 已做实盘抓取验证，当前版本可正常运行。
